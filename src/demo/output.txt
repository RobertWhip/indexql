── 4. Comparison Summary ──────────────────────────────────────

  Approach                            3 queries         Per query avg
  ──────────────────────────────────────────────────────────────────────
  PostgreSQL (indexed, local conn)    ~3.4 ms           ~1.1 ms
  IndexQL (local, after init)         ~5.9 ms           ~2.0 ms
  IndexQL init (one-time)             ~77 ms            amortised

  PostgreSQL strengths:
    • Handles writes, transactions, multi-user concurrency
    • Scales to billions of rows with proper partitioning
    • Full SQL: joins, aggregations, complex ad-hoc queries

  IndexQL strengths (read-only catalog use case):
    • No server required — serve artifacts from a CDN
    • Works offline after the first artifact download
    • Zero infrastructure cost for read-only catalog queries
    • Schema-driven: artifact structure matches query contract exactly
    • Sub-10 ms queries with no network round-trip at all
